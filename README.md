# ESP32 多機能環境時計 & 天気予報ステーション
## (4連OLED + ST7789 TFT + AHT10 + PIR + Touch)

このプロジェクトは、ESP32を使用して、4枚のOLEDディスプレイと1枚の大型TFTディスプレイを組み合わせた多機能な情報表示デバイスです。NTPサーバーからの正確な時刻、OpenWeatherMap APIからの天気予報、およびAHT10センサーによるリアルタイムな室内環境データを統合して表示します。

## 🌟 主な機能

- **5画面同時表示**:
  - **OLED 1**: 日付、曜日、月齢数値、および月相のグラフィック表示。
  - **OLED 2**: 「昨日の天気」と過去24時間の最高・最低気温（履歴データから算出）。
  - **OLED 3**: 「今日の天気」概況と予想最高・最低気温。
  - **OLED 4**: 「明日の天気」概況と予想最高・最低気温。
  - **TFT (中央)**: 巨大デジタル時計、室内温度、室内湿度、および絶対湿度。
- **[改善] スリープ復帰時の72時間気温グラフ**:
  - 省電力モードから復帰した直後の **20秒間**、OLED 2〜4を使用して過去・現在・未来の気温推移を可視化。
  - **視認性の向上**: グラフの線を太く強調し、上下の温度ラベルを大型化（Font2採用）して読み取りやすくしました。
  - **OLED 2**: 過去24時間の気温履歴。
  - **OLED 3**: 今後24時間の予報気温。
  - **OLED 4**: 24〜48時間後の予報気温。
- **高精度な環境計測**:
  - AHT10センサーを使用し、温度・湿度・絶対湿度（g/m³）を算出。
  - 絶対湿度は健康指標に基づき、色分け（緑：快適、黄：注意、赤：警戒）して表示。
- **[修正済み] 堅牢なバックグラウンド更新**:
  - 30分間隔で現在の天気を取得し、過去24時間分（48個）を蓄積。
  - **完全スリープ対応**: 画面が完全に消灯している間も、内部でWiFi経由のデータ蓄積とNTP時刻同期を継続するように修正されました。
  - 蓄積されたデータは不揮発性メモリ(NVS)に保存され、再起動後も履歴が維持されます。
- **省電力 & スクリーンセーバー**:
  - **1分間無操作**: 全画面が「スターフィールド（流れる星）」スクリーンセーバーへ移行。
  - **2分間無操作**: 全ディスプレイが消灯し、バックライトもオフになります（省電力モード）。
  - **復帰**: 人感センサー（PIR）またはタッチセンサー（GPIO33）の反応で即座に復帰し、グラフ表示を経て通常表示に戻ります。

## 🛠 ハードウェア構成

- **マイコン**: ESP32 (DevKit V1など)
- **ディスプレイ**:
  - SSD1306 0.96インチ OLED (I2C) x 4枚
  - ST7789 240x240 TFT (SPI) x 1枚
- **センサー**:
  - AHT10 温湿度センサー (I2C)
  - HC-SR501 または AM312 PIR人感センサー
  - タッチセンサー (GPIO 33)

## 🚀 セットアップ

このプロジェクトでは、セキュリティのためにWiFi設定やAPIキーを別ファイルで管理しています。

1. `include/secrets.h` ファイルを作成します（`.gitignore` によりGit管理から除外されています）。
2. 以下の内容を `secrets.h` に記述し、ご自身の環境に合わせて設定してください。

```cpp
#ifndef SECRETS_H
#define SECRETS_H

// WiFi設定
const char* SECRET_SSID = "あなたのWiFi_SSID";
const char* SECRET_PASS = "あなたのWiFi_パスワード";

// OpenWeatherMap設定 (https://openweathermap.org/)
const char* SECRET_API_KEY = "あなたのAPIキー";
const char* SECRET_CITY = "CityName,CountryCode"; // 都市名,国コード

#endif
```

3. PlatformIOでプロジェクトを開き、ビルドおよび書き込みを行ってください。
