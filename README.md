# ESP32 多機能環境時計 & 天気予報ステーション
## (4連OLED + ST7789 TFT + AHT10 + PIR + Touch)

このプロジェクトは、ESP32を使用して、4枚のOLEDディスプレイと1枚の大型TFTディスプレイを組み合わせた多機能な情報表示デバイスです。NTPサーバーからの正確な時刻、OpenWeatherMap APIからの天気予報、およびAHT10センサーによるリアルタイムな室内環境データを統合して表示します。

## 🌟 主な機能

- **5画面同時表示**:
  - **OLED 1 (0x3C / Bus 0)**: 日付、曜日、月齢数値、および月相のグラフィック表示。
  - **OLED 2 (0x3D / Bus 0)**: 「昨日の天気」と過去24時間の最高・最低気温（履歴データから算出）。
  - **OLED 3 (0x3C / Bus 1)**: 「今日の天気」概況と予想最高・最低気温。
  - **OLED 4 (0x3D / Bus 1)**: 「明日の天気」概況と予想最高・最低気温。
  - **TFT (SPI)**: 巨大デジタル時計、室内温度、室内湿度、および絶対湿度。
- **スリープ復帰時の気温変化グラフ (72時間分)**:
  - 省電力モードから復帰した直後の **20秒間**、OLED 2〜4を使用して過去・現在・未来の気温推移を可視化。
  - **[NEW] インテリジェントなラベル表示**: グラフの高さスケーリング（Y軸）は3枚で共通化しつつ、ラベルには各区間ごとの実際の最高・最低気温を表示するように改善。
  - **OLED 2**: 過去24時間の気温履歴。
  - **OLED 3**: 今後24時間の予報気温。
  - **OLED 4**: 24〜48時間後の予報気温。
- **高精度な環境計測**:
  - AHT10センサーを使用し、温度・湿度・絶対湿度（g/m³）を算出。
  - 絶対湿度は健康指標に基づき、色分け（緑：快適、黄：注意、赤：警戒）して表示。
- **堅牢なデータ保存とバックグラウンド更新**:
  - 30分間隔で現在の天気を取得し、過去24時間分（48個）を蓄積。
  - 蓄積されたデータは不揮発性メモリ(NVS)に保存され、再起動後も履歴が維持されます。
  - 消灯（スリープ）中も内部でWiFi経由のデータ蓄積とNTP時刻同期を継続。
- **省電力 & スクリーンセーバー**:
  - **1分間無操作**: 全画面が「スターフィールド（流れる星）」スクリーンセーバーへ移行。
  - **2分間無操作**: 全ディスプレイが消灯し、バックライトもオフ（ディープスリープではなく、バックライトオフ＋OLEDスリープ）。
  - **復帰**: 人感センサー（PIR）またはタッチセンサー（GPIO 33）の反応で即座に復帰し、グラフ表示を経て通常表示に戻ります。

## 🛠 ハードウェア構成

- **マイコン**: ESP32 (DevKit V1など)
- **ディスプレイ**:
  - SSD1306 0.96インチ OLED (I2C) x 4枚
    - **I2C Bus 0 (SDA:21, SCL:22)**: アドレス 0x3C, 0x3D
    - **I2C Bus 1 (SDA:17, SCL:16)**: アドレス 0x3C, 0x3D
  - ST7789 240x240 TFT (SPI) x 1枚
    - **SPI (MOSI:23, SCLK:18, DC:2, RST:4, BL:5)**
- **センサー**:
  - AHT10 温湿度センサー (I2C Bus 1)
  - PIR人感センサー (GPIO 14)
  - タッチセンサー (GPIO 33)

## 🚀 セットアップ

このプロジェクトでは、セキュリティのためにWiFi設定やAPIキーを別ファイルで管理しています。

1. `src/secrets.h` ファイルを確認（または作成）します。
2. 以下の内容を `secrets.h` に記述し、ご自身の環境に合わせて設定してください。

```cpp
#ifndef SECRETS_H
#define SECRETS_H

const char* SECRET_SSID = "あなたのWiFi_SSID";
const char* SECRET_PASS = "あなたのWiFi_パスワード";
const char* SECRET_API_KEY = "あなたのOpenWeatherMap_APIキー";
const char* SECRET_CITY = "Tokyo,JP"; // 都市名,国コード

#endif
```

3. PlatformIOでプロジェクトを開き、ビルドおよび書き込みを行ってください。

## 📝 最近の更新
- **気温グラフの改良**: 共通スケーリングを維持しつつ、各区間の極値を表示するよう改善。
- **個人情報の隔離**: WiFiのSSID/パスワードやAPIキーを `secrets.h` に分離。
- **バックグラウンド処理の修正**: スリープ中（消灯中）のデータ取得・蓄積を安定化。
- **視認性の向上**: グラフの線の太さを2倍にし、温度表示フォントを拡大。
